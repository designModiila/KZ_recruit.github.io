상세규칙

공통

*네이밍 규칙

1. 내부 퍼블리셔는 카멜케이스를 기본원칙으로 하고, 외부 플러그인으로 인한 클래스는 예외로 적용한다

예시 설명 비고
gnbHeader '-','_'를 사용하지 않고 이어지는 단어의 첫글자를 대문자로 사용한다 카멜케이스(Camel Case)
fp-auto-height 모든 단어를 소문자로 사용하고, 단어와 단어 사이는 바(-)로 연결한다 케밥 케이스(Kebab Case)

2. 네이밍의 시작은 영문 소문자로만 시작한다

잘못된 예시 올바른 예시
Btn btn
1btn btn1
_btn btn
btn_ btn

HTML

*html 구조에 따른 주요 class 네이밍 규칙

  1. 상단 머리글 부분은 header 태그, .header 클래스명을 사용하였으며, 메뉴 부분은 nav 태그를 사용하고 .gnbHeader 클래스명을 사용한다
  2. 중간 컨텐츠 영역은 각 영역에 .section 클래스명을 공통으로 사용한다
  3. 하단 바닥글 부분은 .footer 클래스명을 사용한다

*HTML 파일명 규칙

  1. '서비스영문메뉴이름.html'을 원칙으로 한다 (예: main.html)
  2. 추가적인 설명이 필요한 경우 '서비스영문메뉴이름'뒤에 언더바(_)를 붙여서 작성한다 (예: main_mobile.html)
분류 파일명 페이지명
HTML main.html 고려아연 채용 메인페이지
list.html 고려아연 채용 > 채용정보 > 채용공고 페이지

*전체 폴더구조

폴더분류 폴더명 및 파일 해당 폴더 및 파일에 대한 설명
html main.html/recruit 각 메뉴의 html 또는 해당 html이 두개 이상일 경우 해당 이름의 서비스 폴더를 생성하여 관리한다
assets css/fonts/images/js/plugin css,폰트,이미지리소스,js,plugin 등으로 나눠져있다
images 안에는 common, main, recurit 등 공통 리소스 폴더와 해당 메뉴의 이미지리소스 폴더로 나뉜다

*폴더구조

  1. html/main.html : 고려아연 채용 메인페이지
  2. html/recruit/list.html : 고려아연 채용 > 채용정보 > 채용공고

CSS

*CSS 작성 규칙

1. 기본적으로 한가지 스타일은 한 줄에 작성하며, 줄바꿈 시 빈 줄이 생기지 않게 한다

2. transition, transform 관련 내용은 줄바꿈하여 사용할 수 있다

3. 해당 내용이 어떤 부분의 css인지 주석(예시: /* ----- 비지니스 파트 소개 ----- */)으로 설명해두도록 한다

4. 폰트 단위의 사용은 em으로 작성한다

*폴더구조

  1. asset/css/animation.css : 애니메이션 관련 CSS
  2. asset/css/font.css : 폰트 관련 CSS
  3. asset/css/layout.css : 레이아웃 관련 CSS (Common과 같은 개념)
  4. asset/css/reset.css : 스타일 초기화 관련 CSS

*스크립트 제어 클래스 명

  1. active, on : 선택된 엘리먼트

*CSS 속성 명 정렬 순서

  • 1. display
  • 2. float
  • 3. position
  • 4. z-index
  • 5. top
  • 6. right
  • 7. bottom
  • 8. left
  • 9. width
  • 10. height
  • 11. margin
  • 12. padding
  • 13. 기타 속성
